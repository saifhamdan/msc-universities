# MSc Universities Research

This repository documents my search for a master's degree in computer science or a closely related IT program, with Germany as the current main target. It is meant to be a practical research base, not just a note dump: each country can have its own overview, each university can have its own profile, and my personal assessment stays separate from factual research.

## Why this repository exists

I already work at a strong practical level as a backend and systems engineer, but I want an MSc that adds deeper theoretical grounding in computer science, algorithms, distributed systems, and academic thinking. This repository helps me compare universities in a structured way and build a stronger application story over time.

## Research principles

- Keep the repository markdown-first and easy to browse.
- Separate factual university information from my own opinion and status.
- Prefer official university and admissions pages when adding links or confirming details.
- Record uncertainty honestly instead of pretending missing data is confirmed.
- Make the structure reusable for other countries later.

## Repository structure

## How to read the notes

Each university note follows the same logic:

1. What the university is broadly known for.
2. What matters about the city and living context.
3. Facts captured so far about tuition, timing, language, and availability.
4. Official starting points for further verification.
5. My current take, concerns, and next questions.

## Current scope

The first country fully documented here is Germany because it matches most of my priorities:

- relatively low tuition at public universities,
- strong reputation in engineering and computer science,
- English-taught master's options,
- a realistic path for someone with strong industry experience who wants more theory.

## What this is not

This repository is not a final recommendation engine and it is not a polished admissions package yet. It is a working research system that will eventually feed into better shortlists, stronger motivation letters, and cleaner application planning.

## Next expansion path

When another country becomes relevant, the same structure should be reused:

- one country overview file at the repository root,
- one country folder for university notes,
- the same template for university research,
- the same split between facts and personal assessment.
